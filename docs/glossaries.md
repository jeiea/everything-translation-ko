# 용어집

## 포함 대상

- 기본적으로 여러번 등장하고 번역 일관성이 중요한 용어
  - 고유 기능 이름: 이름만 보고 어떤 기능인지 알기 힘들 수 있어 별도 기록
  - 외래어와 순화표현 둘 다 있는 용어: 섞어 사용하는 혼란 방지를 위해 기록
- 번역 작업 이해에 도움이 되는 정보
  - 번역 문자열 키 패턴: 번역 키만으로 어느 UI에 사용되는지 알기 힘들 수 있어 이해에 도움이 되는 경우 기록

## `_HELP` 접미사

설정 툴팁 텍스트에 붙는 경우가 많습니다.

## Filter: 필터

아래 책갈피와 함께 미묘한 기능이라 [제작자 설명](https://www.voidtools.com/forum/viewtopic.php?p=56913#p56913)을 첨부합니다.

> 한 번에 하나의 필터만 활성화할 수 있습니다.<br />
> 필터는 UI의 검색 설정을 변경하지 않습니다.<br />
> 검색을 다른 검색과 결합하고 싶을 때 필터를 사용하세요.<br />
> 예: ‘폴더만’ 필터를 적용하고 메인 검색 창에 “Program Files”를 입력합니다.

## Bookmark: 책갈피

> 책갈피를 열면 UI의 검색 설정이 변경됩니다.<br />
> 이전 검색을 복원하고 싶을 때 책갈피를 사용하세요.<br />
> 검색을 결합하는 것은 여전히 가능하며, 기존 검색에 내용을 추가하기만 하면 됩니다.<br />
> 검색 조건을 약간 조정하고 싶을 때도 유용합니다.<br />
> 예: c:\users\ \temp\ ext:tmp;exe에 대한 검색을 복원

## Exclude: 제외

색인 단계에서 특정 경로/파일을 아예 데이터베이스에서 빼는 설정.

## Omit Results: 결과 생략

색인 재구성 없이 특정 파일/폴더를 검색 결과에서 숨기는 기능. Exclude(제외)와 달리 색인 자체에서 빼는 게 아니라 결과 표시만 생략합니다.

## File list: 파일 목록

Everything에서 .efu 파일로 검색 결과를 내보낼 수 있고 이걸 파일 목록이라고 부릅니다.

## Macro: 상용구

필터·책갈피에 지정하는 검색 상자 약어. `이름:` 형태로 입력하면 해당 필터/책갈피가 적용됩니다.

## Run history / Run count: 실행 기록 / 실행 횟수

Everything에서 파일을 실행한 횟수를 기록하는 기능. 검색 상자에서 엔터를 누르면 가장 많이 실행한 항목을 우선 선택합니다.

## Home

새 검색 창을 열 때 적용할 기본 검색 옵션(대소문자 구분, 정규식 등) 설정 그룹. `HOME_` 접두사 키들이 해당합니다.

## Index: 색인

Everything이 파일 시스템을 스캔해 만드는 내부 데이터베이스. 인덱스라고 표현하는 건 개발자 대상으로만 적합하다고 생각합니다.

### Rebuild: 재생성

재구축은 덜 사용되어 어려운 느낌이 듭니다.

## ETP Server: ETP 서버

Everything Transfer Protocol. Everything 간 네트워크 검색을 위한 자체 프로토콜입니다.

## Search edit, search box: 검색 상자

'검색 칸'이랑 고민했는데 '검색 상자'가 윈도에서 쓰는 용어이고 원문과 가까워보여 골랐습니다.

## journal: 저널

필요할까 싶지만 기술적 뉘앙스 차이가 있어서 (가령 '기록'같이) 순화하지 않고 외래어 그대로 사용합니다.

## filename: 파일 이름

파일명도 있지만 두 글자 절약보다 순우리말 선호 차원에서 정했습니다.

## scan, rescan: 스캔, 재스캔

`탐색`을 사용했는데 `browse/explore` 의미와 혼동 소지가 있어 외래어를 차용합니다.

- `browse`, `explore`: 탐색
- `scan`: 스캔
- `rescan`: 재스캔

## Toolbar: 툴바

'도구 모음'도 있는데 구글 트렌드상 (도토리 키재기지만) 툴바를 더 많이 쓰는 것 같아 골랐습니다.

## RECALL_ON_DATA_ACCESS: 가상 항목

[계층화된 파일](https://learn.microsoft.com/ko-kr/azure/storage/files/storage-files-faq)(tiered file)이라는 번역이 있는데 봐도 이해하기 어려워 다른 표현을 사용합니다. (내용 설정 페이지에서 살짝 잘리는 이유도 있습니다)

## Focus: 포커스

'초점'을 썼다가 일반적이지 않아보여 '포커스'로 변경합니다.
'활성 항목'도 있는데 '한 페이지 아래 항목 활성화'는 선택이랑 헷갈려보여 차용어가 무난했습니다.
